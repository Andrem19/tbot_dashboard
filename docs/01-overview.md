# 01. Обзор проекта

## Назначение

Проект предоставляет read-only интерфейс для наблюдения за состоянием торгового бота:
- текущая позиция (entry/sl/tp, pnl, side, qty);
- рыночные свечи выбранного инструмента;
- агрегированный отчёт (`report`);
- история закрытых сделок (`hist`) в виде heatmap.

## Технологический стек

- Frontend: React `18.3.1`
- Сборка: Vite `^7.0.4`
- График: `lightweight-charts` `4.1.5`
- База данных: Firebase Realtime Database (`firebase` `^12.0.0`)
- Источник рынка: Binance Futures REST + WebSocket
- Деплой: GitHub Pages (`gh-pages`)

## Основные пользовательские сценарии

1. Пользователь вводит параметры (`coin`, `number_candles`, `interv`) и нажимает `Load`.
2. Приложение загружает свечи и подключает WS поток по выбранному инструменту/интервалу.
3. Приложение подписывается на Firebase-путь `dashb`, получает:
   - `pos` (активная позиция);
   - `report` (поля отчёта);
   - `hist` (история сделок).
4. На графике рисуются свечи, уровни позиции и маркеры истории.
5. Внизу отображаются:
   - полоса отчёта;
   - тепловая карта по дням;
   - статус DB/WS + ручной `Reconnect WS`.

## Что важно знать заранее

- Дашборд рассчитан на тёмную тему в стиле TradingView.
- Основная монета в UI сейчас одна (`coins={[settings.coin]}`), но табы поддерживают расширение под несколько инструментов.
- Секундный таймер рядом с PnL показывает, сколько времени прошло с последнего обновления `dashb.pos`.
